# Contributing Guide

Thank you for contributing to this application repository.
This repo contains application source code that runs in production

Please read this guide before making any changes.

# What changes are allowed

- Application source code changes
- Bug fixes and small improvements
  
# What changes are NOT allowed

- Infrastructure or cloud configuration files
- CI/CD pipeline definitions

# How to contribute

1. Create a new branch from `main`
2. Make one logical change only
3. Test your change locally
4. Write clear commit messages
5. Open a Pull Request with a clear explanation

# Pull Request rules

- Each PR must solve one problem
- Description must explain **what** and **why**
- PR must be reviewed before merging

# Why these rules matter

Mixing infrastructure or secrets with application code can cause security issues.
Following these rules keeps the system safe and stable.
