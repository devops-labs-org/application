# Security Policy

Security is important for this application repository.
Please read this document before reporting any issues.

---

## Reporting a security issue
If you find a security vulnerability, do not create a public issue or pull request.

Instead, report the issue privately
to the repository administrators or maintainers.

---

## What to include in a report
- Description of the security issue
- Affected files or features
- Steps to reproduce the issue
- Any known impact on users or data

---

## What not to do
- Do not share security issues publicly
- Do not add secrets or credentials to the repository

---

## Response process
The maintainers will review the report, work on a fix, and release updates when ready.
