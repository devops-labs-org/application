# Organization Governance Document

# Organization Purpose

This repository is used only for application development.  
All features, bug fixes, and improvements related to the application must be done here using proper GitHub practices like branches and pull requests.

Following these rules helps the team work safely, avoid mistakes, and maintain production-ready code.  
Every member must follow the governance rules mentioned below.

---

# Admin Role Responsibilities

- Create and manage repositories inside the organization  
- Add or remove members from the organization  
- Create and update branch protection rules  
- Review and approve important pull requests  
- Ensure governance rules are followed  

---

# Developer Role Responsibilities

- Create a new branch for every change  
- Do not push code directly to the main branch  
- Raise pull requests for all changes  
- Follow coding and commit message standards  
- Fix issues reported during code review  

---

# Who Can Merge to Main Branch

- Only Admins can merge code into the main branch  
- Developers must raise a Pull Request  
- At least one Admin approval is required before merging  

---

# Who Can Change Branch Rules

- Only Admins are allowed to change branch protection rules  

---

# Rule Violations and Actions

- Direct push to main branch is strictly not allowed  
- If rules are violated, the pull request will be rejected  
- Repeated violations may lead to access removal  
- Admin will review and take action based on severity  

---

#These rules help keep the application safe, stable, and production ready.
